# EternalsMC Dashboard

EternalsMC Dashboard is a comprehensive web application designed for managing a motorcycle club (MC) within a gaming environment. It provides role-based tools for financial management, member tracking, inventory control, and activity coordination. The dashboard features a luxury-themed, responsive interface built with vanilla JavaScript and powered by a Supabase backend.

## Key Features

*   **Role-Based Access Control**: Separate views and permissions for `Admin` and `Member` roles. Admins have full access to modify data, while members have read-only access for most features.
*   **Financial Management**: Track club funds, categorized into "Clean" and "Dirty" money. Admins can log all income and expense transactions.
*   **Member Management**: Admins can add, edit, and remove member profiles, which are displayed in a centralized list.
*   **Armory System**: A detailed inventory management system for items like armor, weapons, ammo, and blueprints. All data is stored locally in the browser, with admins having editing privileges.
*   **Activity Tracking**:
    *   **Daily Checklist**: A list of daily tasks that members can check off to track their activity. Progress resets daily.
    *   **Weekly Quota**: Admins can set a weekly quota, and members' completion can be tracked.
*   **Information Hub**: A built-in price list for various in-game items, helping members with trade and economics.
*   **Dynamic Dashboard**: The main dashboard provides an at-a-glance overview of key stats, including total funds, number of deals, active members, and the current radio channel.

## Technology Stack

*   **Frontend**: HTML5, CSS3, Vanilla JavaScript
*   **Backend**: Node.js (Vercel Serverless Functions)
*   **Database**: Supabase
*   **Deployment**: Intended for Vercel

## Pages & Functionality

The application is structured into several pages, accessible via the sidebar:

*   **Login**: A secure login page to access the dashboard.
*   **Dashboard**: The landing page, displaying key metrics and quick-action tiles.
*   **Deals (Admin)**: Allows admins to add new income or expense transactions to the club's ledger.
*   **Bank (Admin)**: Enables admins to directly deposit or withdraw clean and dirty money from the club's funds.
*   **Checklist**: A list of daily operational tasks for members to complete. Progress is saved in the user's browser.
*   **Quota**: Displays the weekly quota set by an admin and allows members to check off their completion.
*   **Price List**: A reference page showing item prices from different groups/gangs.
*   **Armory**: A detailed view of the club's inventory. Admins can add, edit, or remove items.
*   **Members**: Shows a list of all club members. Admins have controls to add, edit, or remove members from this page.
*   **Transactions (Admin)**: A complete, reversed-chronological log of all financial transactions.

## Setup and Configuration

This project is designed to be deployed on Vercel and connected to a Supabase project.

### 1. Supabase Setup

1.  Create a new project on [Supabase](https://supabase.com/).
2.  In the SQL Editor, create the required tables:

    ```sql
    -- Funds table (stores clean and dirty money)
    CREATE TABLE funds (
      id BIGINT PRIMARY KEY,
      clean BIGINT DEFAULT 0,
      dirty BIGINT DEFAULT 0
    );
    -- Insert a single row for tracking funds
    INSERT INTO funds (id, clean, dirty) VALUES (1, 0, 0);

    -- Members table
    CREATE TABLE members (
      id UUID PRIMARY KEY,
      name TEXT NOT NULL,
      rank TEXT NOT NULL,
      created_at TIMESTAMPTZ DEFAULT NOW()
    );

    -- Transactions table
    CREATE TABLE transactions (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      type TEXT, -- "income" or "expense"
      direction TEXT,
      delta NUMERIC,
      amount NUMERIC,
      balance_after NUMERIC,
      description TEXT,
      created_at TIMESTAMPTZ DEFAULT NOW()
    );
    ```
3.  Navigate to **Project Settings > API** to find your `Project URL` and `service_role` key.

### 2. Vercel Deployment

1.  Clone the repository.
2.  Deploy the project to Vercel.
3.  In your Vercel project's settings, add the following Environment Variables:
    *   `SUPABASE_URL`: Your Supabase project URL.
    *   `SUPABASE_SERVICE_ROLE_KEY`: Your Supabase `service_role` secret key.

Vercel will automatically detect and deploy the serverless functions located in the `/api` directory.

### 3. Local Development

1.  Clone the repository.
2.  Install the Vercel CLI: `npm install -g vercel`.
3.  Create a file named `.env` in the root directory and add your Supabase credentials:
    ```
    SUPABASE_URL=your_supabase_url
    SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
    ```
4.  Run the project locally using the Vercel CLI:
    ```
    vercel dev
    ```
